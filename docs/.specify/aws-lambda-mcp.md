
**Title: Best Practices for Specifying Functions in an AWS Lambda Tool MCP Server**

**Introduction:**

When specifying functions in an AWS Lambda Tool MCP server, it is important to follow a set of best practices to ensure that the functions are well-defined, easy to understand, and maintainable. This document outlines a set of best practices for specifying functions in an AWS Lambda Tool MCP server.

**Best Practices:**

*   **Use a consistent naming convention:** All function names should be in `snake_case`.

*   **Use a clear and concise function description:** The function description should clearly and concisely describe what the function does.

*   **Use a consistent format for the function signature:** The function signature should include the function name, a list of parameters, and the return type.

*   **Use a consistent format for the function parameters:** Each parameter should have a name, a type, and a description.

*   **Use a consistent format for the function return type:** The return type should be a JSON object with a single key, `result`, that contains the function's return value.
